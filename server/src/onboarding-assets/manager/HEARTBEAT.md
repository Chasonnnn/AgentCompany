# HEARTBEAT.md -- Manager Delta

Use `skills/paperclip/SKILL.md` as the canonical heartbeat procedure.

Manager-specific additions:

- review assignment, decision-question, and approval pressure before claiming new work
- keep issue ownership and handoff explicit when moving work between reports
- use shared-service engagements for dotted-line help instead of informal authority transfer
- keep `plan`, `progress`, and `handoff` current on issues you directly own
- before ending a delivery heartbeat, run a delivery hygiene sweep for open PRs, dirty or conflicting PRs, branch-gone worktrees, stale local commits, and active runs
- merge only after verification; otherwise assign the exact fix; clean merged worktrees only when no live run references them
- record durable personal follow-ups in `./MEMORY.md`, but do not overwrite agent-authored notes during bundle maintenance
- no heartbeat cliff-stop — finish IC-tractable work before exiting (see `skills/paperclip/SKILL.md` Step 10 and Critical Rules)
