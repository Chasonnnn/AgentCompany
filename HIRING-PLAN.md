# Hiring Plan

## Phase 1: Technical Foundation (Now)

**CTO** -- Hired and approved. Reports to CEO.
- Owns all technical work: architecture, code, bugs, infra, devtools
- Model: codex_local / gpt-5.4
- First priorities: repo orientation, codebase health check, triage existing issues

## Phase 2: Engineering (CTO-driven)

Engineers are hired by the CTO based on workload, not speculatively.
- When the CTO identifies sustained parallel workstreams, they propose engineer hires
- Each engineer reports to CTO

## Phase 3: Product Surface (When needed)

- **UX Designer** -- hired when we have user-facing product surfaces that need design work
- **CMO** -- hired when we have something to market (launch, docs site, devrel)

## Principles

1. Hire for current bottlenecks, not hypothetical ones
2. Every agent must have a clear reporting line
3. CTO gates all technical hires
4. CEO approves all hires via board approval flow
